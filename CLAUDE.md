# CLAUDE.md — nantotonomemo

このリポジトリで作業する Claude / エージェント向けの運用ガイド。
「何を」「どのファイルで」「どのルールで」やるかをまとめる。
設計の "なぜ" は `DESIGN.md`、日次の元仕様は `prompt.md` を参照。

## これは何か

手持ち無沙汰な時間にできる行動を毎朝自動提案する、個人用の「粗いデジタルツイン」。
`log.md`(履歴)と `profile.md`(人物像)を蓄積し、好み・価値観を **根拠つきで** 推定していく。

## アーキテクチャ: 2層(役割を混ぜない)

- **日次=採取層**: `prompt_cron.md` / 毎朝3時 JST。
  前回 `today.html` の結果(done / 反応 / コメント)を `log.md` に記録し、
  新しい5提案で `today.html` を上書きする。保守的に振る舞う(1件で断定しない)。
- **週次=統合層**: `prompt_weekly.md` / 木曜 01:30 JST(`30 1 * * 4` JST = UTC `30 16 * * 3`、使用量リセット直後)。
  `log.md` 全体を読み、`profile.md`(確定人格)を根拠つきで更新し、`persona_log.md`
  (週報=仮説プール + 変更監査)へ追記する。`today.html` には一切触れない。

ルーティンを動かすときは、必ず対応するプロンプト(cron / weekly)の手順に **そのまま** 従う。
迷ったら「役割を混ぜない」を優先する(採取と統合、today.html と log.md を混同しない)。

## 絶対ルール(hard rules)

1. **出力言語**: 生成物(`today.html` / `log.md` / `profile.md`)は必ず日本語。
   プロンプトが英語でも、生成される中身に英語を混ぜない。
2. **捏造禁止**: `profile.md` の各記述は `log.md` の日付(根拠)に紐づける。
   1件だけで断定せず、複数日・複数の言い方で再現して初めて確信度を上げる。
   存在しないログ日付や架空の提案を作らない。
3. **テンプレート固定**: `today_template.html` のレイアウト / CSS / スクリプトは変更しない。
   差し替えてよいのは 日付 / status / 5タスク / textarea のみ。項目数も変えない。
4. **反応の扱い**: 😕 でジャンルや形式を避けたり頻度を下げたりしない(その回のお題が
   合わなかった程度に扱う)。1回のコメントでジャンル/テーマを丸ごと除外しない
   (繰り返し出て初めて強いルールに格上げ)。
5. **トーン**: 生成物にユーザーへの語りかけ・励まし・説教・絵文字(UI部品以外)を入れない。

## ファイルの役割(どれを触るか)

| ファイル | 役割 | 書き換え |
|---|---|---|
| `prompt.md` | 日次の元仕様(JP) | しない(仕様の出典) |
| `prompt_cron.md` | 日次ルーティンの運用プロンプト(採取層) | しない(運用時は従う) |
| `prompt_weekly.md` | 週次の統合プロンプト(統合層) | しない(運用時は従う) |
| `today.html` | 今日の提案。ユーザーが見る1枚 | 日次のみ上書き |
| `today_template.html` | 固定デザインテンプレート | しない |
| `onboarding.html` | 初回診断フロー(15問) | しない |
| `log.md` | 過去結果の蓄積(1日1行の構造化) | 日次のみ末尾追記 |
| `profile.md` | 安定的な人物像(根拠つき・確定人格のみ) | 日次(明示要望)/ 週次(統合) |
| `persona_log.md` | 週次の人格ログ(週報・仮説プール・変更監査) | 週次のみ追記専用 |
| `DESIGN.md` | 設計思想・決定の背景 | 設計を変えたときのみ |

## 読み込み順(新しいセッションで作業を始めるとき)

1. この `CLAUDE.md` → 2. `DESIGN.md`(なぜ) →
3. 動かす対象のプロンプト(`prompt_cron.md` か `prompt_weekly.md`) →
4. `profile.md` / `log.md`(現状)。週次なら `persona_log.md`(前回の週報・仮説プール)も。
