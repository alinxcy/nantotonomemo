# nantotonomemo

手持ち無沙汰になった時にできる行動を毎朝自動生成するリポジトリ。

毎朝3時(JST)に Routine が起動し、`prompt.md` の指示に従って
`today.html`(今日の提案)を生成し、`log.md`(履歴)と `profile.md`
(人物像)を更新する。

## ファイル構成

- `prompt.md` — 毎朝実行されるメインロジック
- `today_template.html` — today.html の固定デザインテンプレート
  (中身の差し替えのみ許可、レイアウトは変更しない)
- `onboarding.html` — 初回のみ表示される診断フロー(15問)
- `log.md` — 日々の実行結果を蓄積するログ(無期限)
- `profile.md` — 安定的な人物像。診断結果や明示的な要望を蒸留して保持
- `prompt_cron.md` — 日次ルーティンの運用プロンプト(採取層)
- `prompt_weekly.md` — 週次の人格棚卸しプロンプト(統合層。木曜1amリセット直後に実行)
- `DESIGN.md` — 設計思想・主要な決定と背景のメモ

## 運用

1. 初回のみ、`onboarding.html` を開いて診断に回答し、結果を `profile.md` に反映
2. 以後は毎朝 `today.html` が自動更新される
3. チェックボックス・反応ボタン(😊😐😕)・任意コメントを操作するだけでよい
4. 操作しなくても、翌朝のRoutineが前日の状態を自動でログに記録する

### デジタルツイン化(上位ゴール)

log.md / profile.md を蓄積し、ユーザーの「なんちゃってデジタルツイン」を作る。

- **予測→自己採点ループ**: today.html 生成時に profile を使って「本人が各タスクに
  どう反応するか」を先に予測して log に記録 → 翌日、実際の反応と照合してスコア化 →
  外した所を profile の弱点として重点更新する。ツイン再現度を数値で可視化し、
  自動改善につなげる(profile がある程度育ってから着手)。
- **入力ソースの拡張**: 現状は「手持ち無沙汰の過ごし方」の断片のみ。Gmail /
  Google カレンダー連携で生活パターン・関心の信号を足す(プライバシー範囲は要検討)。
- **信号採取の工夫**: コメント欄が最も濃い人格ソース。催促はしない設計を保ちつつ、
  5枠のうち時々1つを「記録系の軽い内省タスク」にして自然にコメントを引き出す。
- **根拠つき人格の徹底**: profile.md の各記述は log.md の証拠(日付)に紐づける。
  1件だけで断定せず、再現して初めて確信度を上げる(捏造・過度な一般化の防止)。
